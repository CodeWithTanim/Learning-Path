### 1. Version Control: Git & GitHub
I learned that developers don't just save files; they use **Git**.

*   **Git:** An open-source tool for tracking changes in code and collaborating with others.
*   **GitHub:** A site (owned by Microsoft) where we host our Git repositories online.
*   **The Difference:** Git is the tool on my computer; GitHub is the cloud storage for my projects.
*   **GitHub Desktop:** A simple app I can use to "Sync" or "Push" my code to the cloud without using complex commands.

---

### 2. Making VS Code Awesome (Extensions)
To make my coding life easier (and cooler), I installed these extensions:

*   **VS Code Icons:** This gives every file a specific icon (like a tiny HTML logo or JS logo). It makes the sidebar look much better!
*   **Jellyfish / GitHub Dark Theme:** I found out I can change the entire look of VS Code. I'm leaning towards the Jellyfish theme—it looks killer!
*   **Live Preview (by Microsoft):** This is a game-changer. I don't have to keep refreshing my browser anymore. I can see my website update *as I type* right inside VS Code.

---

### 3. Creating My First "Linked" Website
I finally learned how to connect the "Skeleton," "Beauty," and "Brain" together. I created three files in my project folder:
1. `index.html`
2. `style.css`
3. `script.js`

#### **How I linked them:**
*   **Linking CSS:** Inside the `<head>` of my HTML, I used the magic `link:css` shortcut to add:
    ```html
    <link rel="stylesheet" href="style.css">
    ```
*   **Linking JavaScript:** Just before the `</body>` tag, I used `script:src`:
    ```html
    <script src="script.js"></script>
    ```

> [!NOTE]
> **Why put Script at the bottom?**
> The teacher said it's a best practice to put the script at the end of the body so the HTML loads first and the website doesn't feel slow.

### 4. Real-World Logic: The iPhone Example
The teacher used pre-ordering an iPhone 15 Pro Max to show how JS works:
*   **HTML:** Shows the text and buttons.
*   **CSS:** Makes the "Titanium Black" color look sleek.
*   **JavaScript:** When I click "Place Order," JS talks to Apple's servers in the background *without* reloading the whole page. This is what makes websites feel "alive."
